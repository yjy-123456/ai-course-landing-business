# 成本与部署指南

## 推荐的第一版架构

使用：

- React + Vite 静态页面
- Cloudflare Pages 或 Vercel 部署
- 飞书表单 / 金数据 / 腾讯问卷收集线索
- 微信二维码承接咨询
- 不做后台数据库

原因：

- 成本低
- 交付快
- 维护少
- 边界清楚
- 非技术客户也容易理解

## 成本预估

自己的 demo：

- 代码：0 元
- 部署：免费额度可覆盖，0 元
- 表单收集：免费版或基础版可覆盖，0 元
- 域名：可选，约 30-100 元/年，具体取决于服务商和后缀

客户项目：

- 如果需要域名，建议让客户提供自己的域名。
- 如果客户没有域名，可以先使用部署平台提供的访问地址。
- 如果客户需要中国大陆服务器和自定义域名，要提醒备案要求。

## 无后台版本适用场景

适合这些情况：

- 客户只需要一个报名页。
- 客户只需要收集线索。
- 客户接受在表单工具里查看报名数据。
- 客户不需要登录、后台或支付功能。

线索收集工具：

- 飞书表单
- 金数据
- 腾讯问卷
- 麦客 CRM
- Tally（能稳定访问时再考虑）

## 什么时候增加后台

只有客户明确需要以下能力时，再考虑增加后台：

- 管理后台
- 数据库
- 课程内容管理
- 支付记录
- 学员管理
- 短信或邮件自动化
- 多角色权限
- 高级数据统计

建议升级报价：

- 简单后台/数据库：3999 元起
- 复杂业务流程：9999 元起

## 给客户的解释话术

第一版建议先不做复杂后台，报名数据进入在线表单，你可以直接查看和导出 Excel。这样上线快、成本低，也适合先验证招生效果。后续如果报名量变大，或者需要学员管理、支付、统计，再升级成独立后台。
